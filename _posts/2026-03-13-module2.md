kw is very interesting, saving developers time, the tutorial in general was pretty clear and easy to follow and config. However, I had a few problems during the configuration. As explained before, I'm following those FLUSP tutorials using a machine with Ubuntu 24.04, in the last tutorial at the first moment I followed using a VM image also ubuntu-based with Ubuntu 24.04, however after a few problems about the ssh configuration, i reconfigured my VM to use a Debian based image, for this tutorial, my image now is Debian 12

A few problems that I had in this tutorial, the most relevant one is my PATH that isn't set correct, for example

``` bash
(DEV-LK) laura-arakaki@laura-arakaki-pc:/home/dev_lk/kw$ kw --version 
/home/laura-arakaki/.local/bin/kw: linha 75: /usr/share/kw/lib/kw_include.sh: Arquivo ou diretório inexistente
/home/laura-arakaki/.local/bin/kw: linha 77: include: comando não encontrado
/home/laura-arakaki/.local/bin/kw: linha 109: signal_manager: comando não encontrado
/home/laura-arakaki/.local/bin/kw: linha 109: warning: comando não encontrado
/home/laura-arakaki/.local/bin/kw: linha 294: include: comando não encontrado
/home/laura-arakaki/.local/bin/kw: linha 296: kworkflow_version: comando não encontrado
(DEV-LK) laura-arakaki@laura-arakaki-pc:/home/dev_lk/kw$ find . -name "kw_include.sh"
./src/lib/kw_include.sh
```

the installation was successful, however the PATH situation didn't work and so I can folow the complete tutorial I adapt the command to add

``` bash
(DEV-LK) laura-arakaki@laura-arakaki-pc:/home/dev_lk/iio$ $KW_DIR/kw <command>
```

I had also other problems, in the command

``` bash
(DEV-LK) laura-arakaki@laura-arakaki-pc:/home/dev_lk/iio$ $KW_DIR/kw ssh --get '~/vm_mod_list'

bash: line 1: rsync: command not found
rsync: connection unexpectedly closed (0 bytes received so far) [Receiver]
rsync error: error in rsync protocol data stream (code 12) at io.c(232) [Receiver=3.2.7]
An error occurred while uploading the file(s). rsync return code: 12
```

I had to run those lines in the @host and @VM

``` bash
apt update
apt install rsync
```

Another observation for the tutorial, the command belloow is very important, however it is a command that takes a lot of time to be executed 

``` bash
kw build
```

One it took about 1 hour to run the command successfully, now i already now that there are a few ways to reduce this time
